# 文章标题优化任务拆分

## 任务概述

将博客中所有"日记型"标题优化为"任务型"标题，提升 SEO 效果和用户体验。

## 任务拆分

### ✅ 阶段 1：分析和准备（已完成）

1. **创建分析脚本** (`scripts/analyze-titles.mjs`)
   - 扫描所有文章标题
   - 识别"日记型"标题模式
   - 检测特殊字符（冒号等）
   - 生成分析报告

2. **分析结果**
   - 总共扫描：199 篇文章
   - 需要优化：63 个标题
     - 日记型标题：44 个
     - 包含特殊字符：18 个
     - 同时存在两个问题：1 个

### 🔄 阶段 2：创建优化脚本（进行中）

1. **创建优化脚本** (`scripts/optimize-titles.mjs`)
   - 将日记型标题转换为任务型标题
   - 移除特殊字符（冒号等）
   - 支持干运行模式（--dry-run）

2. **优化规则**
   - "在 X 中使用 Y" → "Y 完整使用指南 - 如何在 X 中集成并掌握核心功能"
   - "使用 X" → "X 完整使用指南 - 如何快速上手并掌握核心功能"
   - "X 的使用" → "X 完整使用指南 - 如何快速上手并掌握核心功能"
   - "X 实现 Y" → "如何用 X 实现 Y - 完整实战指南"
   - "X 开发 Y" → "如何用 X 开发 Y - 从零到生产完整指南"
   - 移除所有冒号（:）和中文冒号（：），替换为破折号（-）

## 注意事项

1. **特殊字符处理**
   - VitePress 不支持标题中的冒号（:）
   - 所有冒号必须替换为破折号（-）或其他分隔符

2. **SEO 关键词**
   - 任务型标题应包含："如何"、"指南"、"实战"、"完整"等关键词
   - 标题应描述用户能完成的具体任务

3. **标题长度**
   - 建议长度：30-60 个字符
   - 太短：缺少信息
   - 太长：影响可读性

4. **保持一致性**
   - 同一专题的文章标题风格应保持一致
   - 中英文标题应遵循相同的优化原则
